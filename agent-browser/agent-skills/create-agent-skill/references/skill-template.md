# Skill Template

Use this skeleton as the starting point for a workspace-local skill bundle.

```md
---
name: <skill-name>
description: Explain what the skill does and when to use it.
---

# <Skill Name>

## Steps

1. Explain how to start.
2. Explain the most important checks.

## Rules

- Explain the constraints that affect every run.

## References

- Point to deeper docs under references/ when they are needed.
```

Keep the top-level file short enough that the trigger language and workflow are not buried under examples.
