## Design Context

### Users
Primary users are developer-adjacent power users evaluating or using an agent-first browser workspace for research, build, and file-oriented workflows.

Assumption recorded from repo context plus user direction: the interface is used in dense, task-focused sessions where users care more about speed, orientation, and state visibility than brand theater.

### Brand Personality
Fast, dense, restrained.

The interface should feel precise, quiet, and competent rather than friendly, playful, or marketing-led. It should read like a tool for serious work.

### Aesthetic Direction
Modern minimal desktop application UI with a strong editor influence.

- Theme: dark-first
- Primary reference direction: VS Code-like density and information layout
- Explicit preferences: fewer words, fewer oversized controls, minimal surfaces, iconography favored over redundant text
- Anti-direction: card-heavy dashboards, oversized pills, marketing-style hero copy, decorative gradients, soft consumer-app polish

### Design Principles
1. Favor density over spaciousness when it improves scan speed.
2. Replace repeated labels with clear iconography, but keep text where recognition would suffer.
3. Flatten the UI hierarchy: panels, lists, and editors should feel like one workspace, not nested marketing cards.
4. Keep copy terse and utilitarian; every label must earn its pixels.
5. Make navigation state obvious at a glance through alignment, rhythm, contrast, and compact status markers.
