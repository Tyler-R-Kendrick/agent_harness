# Cursor Background Agents Marketing

## Audience

- Professional developers already spending most of their day in an AI-first editor.
- Teams that want background execution but still want review and takeover inside a familiar IDE.
- Engineering orgs evaluating Cursor against Copilot, Claude Code, Codex, Devin, and Windsurf.

## Positioning

- "Asynchronous remote agents in Cursor" rather than a separate outsourcing surface.
- The core promise is flow: start work from the editor, let the agent run remotely, inspect status, follow up, and take over.
- Bugbot positions Cursor as a code-review and quality layer, not only an implementation assistant.

## Customer Model

- Individual and team subscriptions include access to Background Agents.
- Agent usage is charged against model inference pricing, with included usage pools on paid plans.
- Bugbot is a separate review product/add-on in market perception.

## Who They Capture

- Developers who want the shortest path from code context to agent action.
- Teams that want a shared agent environment config checked into the repo.
- Buyers who prefer model-priced usage to enterprise ACU contracts.

## Who They Miss

- Non-engineers who need an app-building workspace rather than an IDE.
- Teams that need browser session replay, visual proof, and human approval checkpoints as first-class artifacts.
- Organizations requiring local-first agent state or self-hosted execution controls.
