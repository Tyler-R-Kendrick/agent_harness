# Browser for AI Agent Marketing

## Audience

- MCP users who want Claude, Codex, Cursor, or custom agents to use their existing browser tabs.
- Developers interested in WebMCP/page tools as a structured alternative to blind browser automation.
- Local-first users who prefer extension plus native host over remote browser sessions.

## Positioning

Browser for AI Agent positions itself as a browser extension that lets AI agents read browser tabs, control the browser, and execute page tools through a local MCP service.

## Customer Model

- Open-source GitHub project with release downloads.
- Chrome Web Store and Firefox Add-ons listings are pending in the current snapshot.
- No visible paid plan; adoption depends on MCP community sharing and extension installability.

## Capture

- Captures local-first users who need agents to work with logged-in tabs.
- Captures developers who want page-native tools and WebMCP to reduce brittle scraping.
- Captures multilingual/open-source users through the repository's English and Chinese README surfaces.

## Miss

- Misses users who require store-reviewed, one-click installation today.
- Misses managed infrastructure buyers needing remote concurrency and session storage.
- Misses teams without a security model for exposing cookies, storage, and arbitrary page scripts to agents.
