# Claude Code Marketing

## Target Customer

- Professional developers who are comfortable in a terminal and want an agent with direct repository access.
- Teams that want coding assistance embedded into existing scripts, Git workflows, CI, and review.
- Power users who will invest in `CLAUDE.md`, skills, hooks, MCP servers, and custom subagents.
- Enterprises that want Anthropic models with controllable permissions and deployable workflows.

## Positioning

- "Agentic assistant that runs in your terminal" rather than a conventional IDE plugin.
- Anthropic positions the tool around a repeatable loop: gather context, take action, verify, and let the human interrupt.
- The extension story is a core part of the brand: Claude Code can be customized into a team-specific software agent.

## Customer Model

- Claude Code usage depends on sign-in mode: included seat pools for plans or pay-as-you-go token billing through Console, Bedrock, Vertex, or Microsoft Foundry.
- Model choice affects usage and cost, with guidance to reserve Opus for harder tasks and use Sonnet or Haiku for lighter work.
- GitHub Actions, code review, and enterprise deployments create a path from individual CLI adoption to organization-wide procurement.

## Who They Capture

- Developers who trust the terminal and want an agent that can run the same commands they do.
- Teams that treat project instructions, hooks, and reusable skills as operational infrastructure.
- Users who value fast iteration over polished visual product surfaces.

## Who They Miss

- Non-developers and teams that need a guided GUI for supervising browser work.
- Users who need rich screenshots, videos, and page-level traces as first-class output.
- Teams with low tolerance for usage-limit uncertainty or terminal permission risk.

## Agent Browser Opportunity

- Match Claude Code's extensibility while making browser authority and evidence easier to inspect.
- Preserve visible page state, screenshots, recordings, approvals, and recovery points alongside any terminal transcript.
- Offer a smoother browser-first review model for users who do not want the browser to be just another MCP tool call.
