# Claude Code Gossip

## What People Praise

- Community praise centers on coding quality, terminal fit, project understanding, and the ability to chain many actions without leaving the repository.
- Power users like `CLAUDE.md`, hooks, MCP, subagents, and skills because they can encode workflow standards.
- Claude Code is often treated as the benchmark other coding agents are compared against.

## What People Complain About

- Usage limits are a persistent complaint, especially when users feel sessions drain faster than expected.
- Some users report perceived model degradation, lazy fixes, or failure to sustain deep reasoning across long tasks.
- Terminal autonomy creates fear when commands are broad, destructive, or insufficiently scoped.
- Browser or visual checks can feel bolted on compared with the richness of the coding loop.

## Bug And Risk Themes

- Rate-limit confusion and context-management fatigue.
- Overconfident fixes that require user correction.
- Permission-prompt fatigue, especially when users relax controls to keep momentum.
- Extension or browser integration security risk when a local agent gets access to sensitive contexts.

## Design Sentiment

- The low-friction terminal UX is the strength and the weakness: it is fast for developers, but it can hide important state in a long stream of text.
- Users want the agent to be autonomous until something goes wrong, then they want precise evidence and reversible recovery.
