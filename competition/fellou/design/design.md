# Fellou Design

## Look And Feel

- AI-native productivity landing page with large headline typography, animated task/status panels, and a strong "browser as assistant" visual metaphor.
- The product story is built around agent work surfaces: running tasks, source gathering, report compilation, editable plans, and real-time intervention.
- Marketing pages use high contrast, oversized claims, and workflow examples that feel closer to an AI operating system than a conventional browser.

## Design Tokens To Track

```yaml
surface: glossy AI productivity marketing
accent: blue/purple/green action cues with globe and task-status motifs
primary_control: natural-language goal prompt with generated action plan
secondary_controls:
  - approve or edit plan
  - intervene during workflow
  - tag groups
  - multi-source report outputs
  - desktop/file permissions
trust_controls:
  - visible step plan
  - user approval before execution
  - local-first/privacy-first claims
information_density: high
```

## Differentiators

- Fellou sells "action" as the primary design object. The browser is a workspace for delegating tasks, not just viewing pages or chatting beside them.
- Plan-first execution is an important UX differentiator because it gives users a checkpoint before the agent starts clicking or using accounts.
- The "spatial" Concept Edition language suggests layered workspaces and asynchronous task groups rather than a single tab/sidebar model.

## What Is Good

- The action-plan review pattern is clearer than black-box autonomous browsing.
- Multi-source visual reports make research output inspectable and shareable.
- The design speaks directly to users who feel like they are manually copying data between SaaS tools.

## Where It Breaks Down

- The density of claims can overwhelm first-time users and make the actual product boundaries hard to understand.
- "Desktop files + browser + memory + logged-in accounts" creates a large trust surface that the UI must explain continuously, not only in FAQ copy.
- Agentic workflows can feel slow or overbuilt for simple tasks; the product needs strong mode switching between direct browsing and delegation.
