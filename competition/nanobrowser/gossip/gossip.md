# Nanobrowser Gossip

## Positive Signals

- GitHub visibility gives the project credibility with developers who want to inspect the agent and extension behavior.
- The existing-browser approach is a strong answer to users who do not want another browser.
- Open-source and BYOK messaging aligns with recurring privacy concerns around hosted AI browsers.

## Negative Signals

- GitHub issues are the main public support signal, which suggests a still-maturing product with community-driven bug discovery.
- Extension agents can break when browser APIs, page structures, or model responses change.
- Chrome Web Store discoverability and user trust are harder to build than a model-owner assistant or incumbent browser feature.

## Bug And UX Risk Themes

- Model setup and provider failures can look like product failures to users.
- Permission prompts need plain-language explanations or privacy positioning will not land.
- Real-profile action requires stronger replay, approval, and secret-handling affordances than most extension UIs show by default.
