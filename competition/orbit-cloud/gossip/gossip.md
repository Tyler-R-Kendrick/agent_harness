# Orbit Cloud Gossip

## Positive Signals

- Official messaging directly addresses common production gaps: runtime parity, reliable auth, orchestration, observability, autoscaling, RBAC, and VPC deployment.
- The local-development-to-cloud-deployment narrative fits how browser automations are actually built and stabilized.
- General browser-agent community discussion supports the need for logs, screenshots, retries, and reliable auth when sites change or sessions expire.

## Negative Signals

- Public third-party chatter specific to Orbit Cloud is limited, so most evidence is official-source.
- Pricing is not public and is described as still being decided, which can make early comparison difficult.
- Enterprise controls may make the product feel heavier than needed for simple local browser-agent tasks.

## Bug And UX Risk Themes

- Runtime parity must be exact, or users will lose trust when local runs pass and cloud runs fail.
- Auth helpers and 2FA flows need careful audit and user consent design.
- Cloud schedules and retries can amplify cost or unwanted actions if guardrails are weak.
