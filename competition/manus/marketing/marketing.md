# Manus Marketing

## Audience

- Prosumer operators who want a general AI agent to research, book, extract, write, and coordinate without staying in the loop.
- Startup and business teams that want a single autonomous assistant for web, document, file, and coding tasks.
- Developers that want to create and manage agent tasks programmatically through the Manus API.

## Positioning

- "Agent that acts" rather than assistant that answers.
- Asynchronous cloud execution for long-running work.
- Local Browser Operator for authenticated websites and sensitive workflows where cloud browsers trigger security checks or require repeated login.

## Customer Model

- Subscription and credit-driven usage for individuals and teams.
- Browser extension install creates a low-friction bridge into the user's existing desktop browser.
- API access opens an automation-platform motion for teams that want Manus as an execution backend.

## Who They Capture

- Users who want one broad agent instead of separate tools for research, app use, file handling, and browser tasks.
- Logged-in workflow users frustrated by cloud browsers, CAPTCHA, and session setup.
- Buyers comfortable with supervised autonomy and post-task review.

## Who They Miss

- Teams that need deterministic test artifacts, repo-local state, or a browser-agent workbench they can inspect and extend.
- Security-heavy buyers that need strict per-domain policies and state-changing action gates before local browser control.
- Cost-sensitive users who need predictable price-per-success before starting long tasks.
