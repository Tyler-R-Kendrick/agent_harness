# Make AI Agents Gossip

## Positive Signals

- Make's official positioning leans into trust, transparency, and cost visibility, which directly answers common AI-agent adoption concerns.
- Help docs expose practical controls such as model selection, max steps, max history, context files, MCP tools, and module tools.
- Community questions about pricing show real buyer interest in how AI Agents will scale beyond beta-style experimentation.

## Negative Signals

- Users ask for clearer pricing because agent runs vary more than traditional Make scenarios.
- Reusable shared agents are powerful but can create coordination risk when a global prompt or setting update affects many modules.
- The platform's strength in business workflows may leave browser-specific debugging, screenshots, and visual state underrepresented.

## Bug And UX Risk Themes

- Shared-agent changes can break scenarios that depend on prior prompt or tool behavior.
- Cost prediction requires sample runs and analysis because agent decisions and token use vary.
- MCP/tool additions expand capability but also expand the permissions and failure surface.
