# Make AI Agents Design

## Look And Feel

- Visual-first automation canvas with AI agents embedded into the same orchestration environment.
- Marketing screenshots emphasize transparency, decision visibility, reusable agents, and workflow-connected tools.
- Help docs describe a configuration surface for model choice, system prompts, context files, MCP tools, and Make modules.

## Design Tokens To Track

```yaml
surface: visual automation canvas plus agent configuration tabs
accent: Make purple and blue product gradients with clean white workspace panels
primary_control: canvas orchestration and run-agent modules
secondary_controls:
  - system prompt
  - model settings
  - maximum steps
  - context files
  - MCP tools
  - Make modules
trust_controls:
  - step-by-step decision visibility
  - reusable team agents
  - usage visibility
  - manual configuration rollback
information_density: high
```

## Differentiators

- Agents are built directly where Make users already create scenarios, reducing the gap between AI experimentation and production workflow.
- Make emphasizes transparent decisions, visible usage, reusable setups, and shared agents across teams.
- MCP support and module tools let agents use both Make's integration ecosystem and external tool surfaces.

## What Is Good

- Strong fit for business users who already think in visual scenarios and app modules.
- Transparent-decision messaging directly addresses trust concerns around AI automation.
- Context files, tools, model settings, and max-step controls give builders practical levers to constrain behavior.

## Where It Breaks Down

- Visual workflow clarity can still become complex when agents, modules, context files, MCP tools, and model settings interact.
- Pricing and operation costs are not as immediately obvious as deterministic scenario runs, especially while agent behavior varies.
- Browser-level evidence is secondary to workflow-level decision visibility.
