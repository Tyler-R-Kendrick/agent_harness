# AgentQL Marketing

## Audience

- Developers building web agents and data workflows.
- Scraping teams that want structured data without hand-maintaining selectors for every page.
- Agent-framework users who need a web interaction tool inside LangChain, LlamaIndex, Dify, Zapier, MCP clients, or similar stacks.
- Teams that want a remote browser option without committing to a full browser-infrastructure vendor.

## Positioning

- Core claim: a query language and developer toolkit that identifies web elements and data by natural-language descriptions and returns the shape the developer defines.
- Competitive frame: more robust and readable than raw selectors, more programmable than manual scraping, and easier to integrate than building a browser agent from scratch.
- The product markets "web agents and data workflows" directly in pricing copy.

## Customer Model

- Free trial with API calls, remote browser time, one concurrent remote-browser session, and developer tools.
- Starter is $0/month with smaller monthly API and browser-hour allowances plus per-call and per-hour overages.
- Professional is $99/month with higher API-call, browser-hour, and concurrency limits.
- Enterprise offers managed solutions, dedicated cloud, on-premise deployment, support, and account management.

## Who They Capture

- Developers frustrated by CSS/XPath selector churn.
- Teams that want the LLM-facing contract to be a compact data/query shape.
- Agent builders who want a semantic locator layer while keeping Playwright and JavaScript/Python control.

## Who They Miss

- Users who want a visible browser-agent workspace with live approvals and run replay.
- Buyers who require deterministic selectors and exact failure modes for regulated workflows.
- Teams that need anti-bot, proxy, CAPTCHA, and browser-fleet operations as the primary product rather than a query layer.
