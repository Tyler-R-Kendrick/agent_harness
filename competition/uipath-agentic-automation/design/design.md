# UiPath Agentic Automation Design

## Look And Feel

- Enterprise suite design: glossy product pages, large outcome claims, industry tiles, and many cross-links into platform modules.
- The product language uses a memorable operating model: agents think, robots do, people lead.
- Documentation and product surfaces emphasize orchestration, governance, guardrails, and reusable platform capabilities more than a single daily browser workspace.

## Design Tokens To Track

## Differentiators

- Combines classic UI automation, API automation, agents, document processing, testing, process discovery, and human work in one suite.
- UiPath Maestro positions orchestration as the missing control plane for agentic AI.
- Healing Agent messaging directly targets brittle UI automation by promising adaptation when interfaces change.
- Autopilot for Everyone turns agentic automation into an employee-facing companion rather than only a developer tool.

## What Is Good

- Strong enterprise trust story: governance, compliance, guardrails, cloud/self-hosting, and CoE controls are visible.
- The split between agents, robots, and people is clearer than products that imply a single model should do everything.
- Existing RPA footprint gives UiPath a large migration path from deterministic bots to agentic exception handling.

## Where It Breaks Down

- The suite can feel heavy for teams that only need inspectable browser-agent runs, local state, or fast developer iteration.
- Product vocabulary spreads across Autopilot, Agent Builder, Maestro, robots, process apps, and healing agents, which can obscure the first workflow to try.
- Community feedback repeatedly worries that agentic automation can be sold into processes where deterministic RPA would be cheaper and easier to audit.
