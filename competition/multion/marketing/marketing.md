# MultiOn Marketing

## Audience

- AI engineers building products that need web actions rather than just text generation.
- Automation teams that want natural-language browser execution without running their own browser fleet.
- Developers who need both isolated cloud sessions and local browser state.

## Positioning

- "Motor Cortex layer for AI" is the main category phrase.
- The product emphasizes autonomous actions on the web, millions of concurrent agents, structured scraping, secure remote sessions, local Chrome extension mode, and proxy support.
- The docs warn that Agent V1 is beta and that outputs should be verified, which is honest but also a trust signal to experienced builders.

## Customer Model

- API-first developer adoption through docs, examples, playground, and SDKs.
- Community/support motion through Discord and email.
- Likely usage-based scaling for teams running many browser-agent sessions.

## Who They Capture

- Builders who want to outsource the browser execution layer but keep their own app UX and orchestration.
- Teams that value screenshots, session IDs, and stepwise recovery more than a consumer AI browser.
- Workflows where proxy support and remote isolation matter.

## Who They Miss

- End users who want a polished assistant workspace rather than an API.
- Teams that need local-first storage, repo-native artifacts, or UI-level process evidence inside an agent workbench.
- Buyers who prefer deterministic scripts when an API exists and view browser agents as a last resort.
