# Axiom.ai Marketing

## Audience

- Operators, growth teams, sales teams, ecommerce users, and analysts who need repetitive browser scraping or data entry without engineering support.
- Small teams that already work in Chrome, Google Sheets, Zapier, Make, and web apps with weak or unavailable APIs.
- Technical users who can extend a no-code bot with JavaScript or the pro-code beta, but do not want to start from a browser automation framework.

## Positioning

- Axiom.ai positions itself as no-code browser automation and web scraping that is quick to build in Chrome.
- The product claims the practical middle ground between manual work and custom code: scrape data, fill forms, move data across apps, schedule jobs, and add AI extraction.
- Privacy messaging emphasizes local desktop processing for local runs, while cloud runs are packaged around runtime quotas and run-duration limits.

## Customer Model

- Free trial gives limited runtime for evaluation.
- Paid tiers sell monthly runtime, cloud concurrency, cloud duration limits, scheduling frequency, integrations, and support level.
- Expansion path is runtime growth: more hours, more concurrent cloud bots, team sharing, API/webhook use, and priority support.

## Captures

- Users who see browser automation as a business operations problem, not an agent research problem.
- Teams that value templates, tutorials, and integrations more than source-level control.
- Workflows where a Chrome extension can reuse normal browser context and avoid server-side API setup.

## Misses

- Engineering teams that need deterministic tests, versioned code review, or full trace artifacts.
- Security-sensitive teams that need explicit approval boundaries for authenticated browser actions.
- Users whose jobs are better served by stable APIs than brittle browser interaction.
