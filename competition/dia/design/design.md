# Dia Design

## Look And Feel

- Dia moves away from Arc's highly opinionated vertical-tab/workspace design toward a familiar Chromium-like browser.
- AI is centered in the URL bar and side/chat surfaces rather than framed as a separate app.
- The product preserves The Browser Company's polished microinteraction/design reputation while lowering Arc's learning curve.

## Design Tokens To Track

```yaml
surface: familiar Chromium browser shell
accent: restrained Browser Company branding
primary_control: URL bar as AI/search/navigation input
secondary_control: tab-aware chat surface
personalization:
  - tone/style preferences
  - coding settings
  - opt-in browsing history context
skills:
  - user-created shortcuts
  - built-in planning/writing/learning/coding tasks
information_density: medium
```

## Differentiators

- Dia's strongest design idea is making the omnibox the AI interface.
- "Mention tabs" and "ask about all open tabs" turns tab state into a first-class context model.
- Skills let users create repeatable browser behaviors without reaching for external automation tools.

## What Is Good

- Familiar browser shell reduces adoption friction compared with Arc.
- Tab-aware AI maps well to actual research workflows.
- Skills are a strong bridge between casual prompting and reusable workflows.

## Where It Breaks Down

- By becoming more familiar, Dia loses some of Arc's distinctive workspace power.
- macOS/Apple Silicon constraints limit browser replacement potential.
- The AI model is only as useful as the user's confidence in history/memory access.
