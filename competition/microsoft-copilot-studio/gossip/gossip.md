# Microsoft Copilot Studio Gossip

## Positive Signals

- Community builders like the ability to create simple agents directly inside Microsoft 365 Copilot Chat and then share them into daily work surfaces.
- Enterprise teams value Microsoft Graph grounding, identity-aware access, and the chance to avoid a separate agent vendor.
- The product is perceived as strategically inevitable because Microsoft controls the productivity suite and Teams distribution.

## Negative Signals

- Repeated community confusion centers on whether an agent was built in Microsoft 365 Agent Builder or full Copilot Studio, and why agents created in one surface do not always appear or behave in another.
- Billing discussions focus on which actions are included in Copilot licenses and which autonomous or Power Automate-triggered runs consume Copilot credits.
- Users report orchestration frustration when sub-agents work individually but lose context, route ambiguously, or fail when chained.

## Bug And Friction Themes

- Publishing and channel visibility can be opaque, especially across Teams, Microsoft 365 Copilot Chat, and Studio.
- Multi-agent state and variable passing require explicit design; otherwise sub-agents forget input or fall back generically.
- Naming churn and overlapping Microsoft surfaces make onboarding harder than the lightweight builder suggests.
