# dassi Gossip

## Positive Signals

- First-party pages claim tens of thousands of Chrome users and high Chrome Web Store ratings.
- The about page gives a credible privacy narrative based on prompts going from the browser to the selected AI provider rather than through vendor routing.
- The docs are straightforward and explain setup, model/provider choices, and the basic plan/execute/verify loop.

## Negative Signals

- The main page and product sections use inconsistent adoption numbers, which makes the growth signal less clean.
- Public third-party review and issue chatter is limited in search results, so the main sentiment signal is first-party.
- "Any task you do in the browser should be delegatable" is broad enough to set expectations beyond what a sidebar agent can reliably guarantee.

## Bug And UX Risk Themes

- Extension permissions and screenshots/page content sent to model providers must remain easy to understand.
- Approval fatigue can appear if every small action needs confirmation, but reduced approval can increase risk.
- Saved skills and scheduled runs need clear edit/history/error UX or they become opaque automations.
