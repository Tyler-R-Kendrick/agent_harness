# SurfAgent Marketing

## Target customer

- Individual developers and agent users running Claude Code, Codex, Cursor, Windsurf, Gemini CLI, Hermes, OpenClaw, or custom MCP/CDP agents.
- Users who want a real logged-in browser for background automation without paying cloud browser subscriptions.
- Coding-agent users who need screenshots, UI verification, frontend debugging, research, scraping, and form workflows.

## Positioning

- "Your AI agent deserves a real browser" is the core frame.
- SurfAgent positions against browser extensions, shared personal browser control, and cloud browser services.
- It sells a dedicated local Chrome profile, full CDP access, MCP integration, REST API, persistent cookies, crash recovery, and $49 lifetime ownership.

## Captures

- Price-sensitive local-first users who dislike browser-hour billing.
- Developers who understand CDP/MCP and want a fast setup path.
- Users whose tasks require logged-in sessions, dynamic pages, screenshots, or agentic frontend QA.

## Does not capture

- Enterprise buyers that need team policy, central audit, managed seats, compliance, and hosted scale.
- macOS/Linux users if the Windows-first app remains the primary distribution.
- Teams that avoid local daemon access to sensitive browser sessions.
