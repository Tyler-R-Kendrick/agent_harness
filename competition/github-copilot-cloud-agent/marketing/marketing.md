# GitHub Copilot Cloud Agent Marketing

## Audience

- GitHub-first engineering teams that already plan work in issues and review through pull requests.
- Enterprises that prefer AI agents governed by existing GitHub policies, audit logs, and licenses.
- Developers who want delegation without leaving GitHub, VS Code, JetBrains, Eclipse, Visual Studio, mobile, CLI, or MCP-enabled tools.

## Positioning

- Copilot is marketed as an AI coding assistant that helps from IDE suggestions through research, planning, code changes, and PR creation.
- The cloud agent reframes Copilot from pair programmer to async contributor.
- Third-party agent support positions GitHub as the control plane for multiple coding agents, not only its own model stack.

## Customer Model

- Available in Copilot Pro, Pro+, Business, and Enterprise plans depending on feature.
- Cloud agent work consumes Copilot premium requests and GitHub Actions minutes.
- Enterprise value comes from policy controls, custom agents, model choices, audit logs, and existing procurement.

## Who They Capture

- Teams already living in GitHub Issues, PRs, Actions, and Copilot licenses.
- Managers who want to assign well-defined issues and return to a PR.
- Organizations that want agent choice through a single GitHub governance layer.

## Who They Miss

- Teams that need local-first browser workbench state, custom UI, or artifact-backed visual regression proof.
- Users who do not trust GitHub-hosted cloud execution with their repo context.
- Builders working outside GitHub or in self-hosted/nonstandard SCM environments.
