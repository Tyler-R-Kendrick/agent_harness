# Sema4.ai / Robocorp Marketing

## Audience

- Python automation developers.
- RPA teams that want code-first automation instead of a low-code suite.
- AI-agent builders that need reliable callable tools for real business systems.

## Positioning

- Sema4.ai frames the product around building AI agents and exposing existing automation as AI actions.
- Robocorp materials emphasize Python, VS Code, RCC, Action Server, and Control Room.
- The implicit pitch is that agents need dependable tools, and Python automation is a good substrate for those tools.

## Customer Model

- Developer-first adoption through docs, CLI, VS Code extension, and local Action Server.
- Enterprise deployment through Control Room, workspaces, secrets, and operator-managed runtime configuration.
- Expansion path from scripts and robots into AI-agent action libraries.

## Captures

- Teams that want code ownership and testable automation.
- Developers who dislike black-box browser agents and prefer explicit Python tools.
- Organizations already invested in Robocorp/RCC.

## Misses

- Nontechnical business users who expect a chat-first browser workspace.
- Buyers looking for a single polished enterprise automation suite.
- Teams confused by the Robocorp/Sema4.ai rebrand or uncertain about product continuity.
