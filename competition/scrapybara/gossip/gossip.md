# Scrapybara Gossip

## Positive Signals

- Scrapybara's docs are current in the public search index and present concrete browser, desktop, auth, stream, and Act SDK capabilities.
- The product maps tightly to the rise of computer-use models, where browser-only tools are not enough.
- The API shapes are legible to developers familiar with Playwright, remote desktops, and model tool loops.

## Negative Signals

- Kernel's migration documentation claims Scrapybara shut down its virtual desktop and browser service on October 15, 2025. Because Scrapybara's own site and docs remain accessible and current-looking, this should be tracked as conflicting market intelligence rather than assumed fact.
- Whole-desktop automation is a broader and less bounded surface, which can make debugging, permissions, and artifact review harder than browser-specific MCPs.
- The category has a history of vendors pivoting or shutting down as browser infrastructure costs and reliability burdens rise.

## Buggy Or Risky Areas

- Vendor continuity and migration planning.
- Cloud desktop latency and cost under high-step agent loops.
- Auth-state handling and credential governance.
- Debugging failed computer-use trajectories without product-native eval artifacts.
