# Chrome DevTools MCP Marketing

## Audience

- Coding-agent users in Antigravity, Claude Code, Copilot, VS Code, Cursor, Gemini CLI, and similar tools.
- Web developers who want agents to inspect real browser state while fixing or testing code.
- Teams that trust official Chrome tooling more than small third-party browser bridges.

## Positioning

Chrome DevTools MCP positions itself as "Chrome DevTools for agents": reliable automation, in-depth debugging, and performance analysis through the Model Context Protocol.

## Customer Model

- Open-source, npm-distributed developer tooling.
- Adoption benefits from Chrome DevTools brand trust, agent-client install snippets, and plugin/skill packaging.
- No direct paid customer model in the repository snapshot.

## Capture

- Captures developers who already debug with Chrome DevTools and want the same authority inside coding agents.
- Captures agent-client ecosystems looking for a canonical browser MCP.
- Captures app-debugging workflows where console, performance, page state, and browser inspection are more important than remote browser fleets.

## Miss

- Misses business operators who need packaged task workflows.
- Misses cloud automation buyers who need concurrency, proxies, identity, replay storage, and team dashboards.
- Misses local-browser products that differentiate on friendly visible workspaces, approvals, and human handoff.
