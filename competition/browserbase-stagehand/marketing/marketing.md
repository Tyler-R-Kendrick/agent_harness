# Browserbase + Stagehand Marketing

## Audience

- Developers building browser agents, scraping workflows, QA agents, data retrieval systems, and automations behind auth walls.
- Startups and enterprises that do not want to maintain browser fleets.
- Teams needing logs/replay/observability for production agent runs.

## Positioning

- Core claim: complete platform for agents that browse and interact with the web like humans.
- Market frame: traditional APIs cover only a fraction of the web; the rest needs browsers, identity, JS rendering, and interaction.
- Product promise: one API key for browsers, search, fetch, model access, functions, identity, and observability.

## Customer Model

- SaaS infrastructure with usage-based browser/session consumption and higher-tier enterprise support.
- Developer adoption via open-source Stagehand and quickstarts.
- Enterprise upsell through scale, security, identity, and support.

## Who They Capture

- Production teams that need reliability and browser fleet management.
- Developers who already use Playwright/Puppeteer/Selenium.
- Teams that view browser automation as infrastructure, not a UI product.

## Who They Miss

- End users who want an AI browser they can personally browse in.
- Local-first users avoiding cloud browser/data processing.
- Users who want no-code UI workflows instead of SDK/API control.
