# Agent Browser Marketing

## Audience

- AI engineers building browser agents into Vercel AI SDK apps.
- Cursor, Claude Desktop, and MCP users who want a compact browser tool.
- Developers sensitive to token cost from screenshots, DOM dumps, or accessibility-tree snapshots.

## Positioning

Agent Browser positions itself as a token-efficient browser for AI agents. The central claim is not managed infrastructure; it is a compact representation that agents can act on cheaply.

## Customer Model

- Open-source or npm-led developer adoption.
- No visible paid plan in the current product surface.
- Distribution depends on GitHub stars, npm installs, SDK examples, and MCP copy-paste setup.

## Capture

- Captures developers who want a small tool surface and easy model integration.
- Captures cost-sensitive agent experiments where screenshots and full DOM snapshots are too expensive.
- Captures workflows where the agent can work from semantic/textual refs rather than human-visible replay.

## Miss

- Misses buyers who need managed sessions, proxies, identity, recordings, and dashboards.
- Misses end users who want a polished local AI browser.
- Misses safety-conscious workflows unless the host app adds approvals, permissions, and durable evidence.
