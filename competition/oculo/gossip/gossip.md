# Oculo Gossip

## Positive Signals

- Oculo's token-minimal pitch lines up with category complaints that DOM snapshots and screenshots make browser agents expensive and noisy.
- The app is more approachable than many MCP browser servers because it looks like a full browser with a recognizable AI side panel.
- Local privacy, no telemetry, PII redaction, and keychain storage address the main fear around agents controlling logged-in sessions.

## Negative Signals

- The product is still young and needs more independent evidence around reliability on complex authenticated workflows.
- Compact page descriptions can create silent failure if the agent misses labels, warnings, or disabled controls.
- Daily-browser positioning means users compare Oculo not only to agent tools but also to polished incumbents like Chrome, Edge, Dia, Comet, and Arc-style workflows.

## Category Chatter

- MCP browser-server discussions repeatedly mention dynamic classes, shadow DOM, prompt injection, and giant snapshots as failure modes.
- Users want browser agents to interact with their real browser state, but they also worry about secrets, unintended clicks, and agent-visible page instructions.
- The market is moving toward visible permission tiers, local execution, and shorter page briefs as table stakes.

## Bug And UX Risks To Watch

- Shell-capable tooling should be visibly scoped away from ordinary browsing tasks.
- Permission tiers need clear per-action copy; otherwise "auto" can become too permissive.
- A browser that promises daily use must keep ordinary browsing fast even when MCP clients are connected.
