# Bright Data Agent Browser Marketing

## Audience

- AI-agent developers whose agents need live public-web access and multi-step browser actions.
- Data teams that already buy proxies, scraping APIs, datasets, or web-unlocking services.
- Enterprises that need managed data acquisition, support, SSO, audit logs, and SLA terms.
- MCP users who want web search, scraping, markdown extraction, and browser automation inside Claude, Cursor, Windsurf, Codex, or custom clients.

## Positioning

- Core claim: a browser built for AI agents that can navigate any public website like a human would.
- Competitive frame: ordinary headless browsers are detectable; Bright Data provides headful/cloud browser infrastructure with unlocking, proxies, CAPTCHA solving, retries, and data extraction.
- MCP pricing reframes the product as an agent gateway to search, scrape, extract, and automate the web.

## Customer Model

- MCP free tier includes a monthly request allowance for testing and light use.
- Paid usage and deposit promotions convert developers into Bright Data accounts.
- Enterprise packages add account management, custom packages, premium SLA, onboarding, SSO, customizations, and audit logs.

## Who They Capture

- Buyers who care most about whether a website can be accessed reliably at scale.
- Data-pipeline teams that want browser automation connected to scraping, structured data, and proxy infrastructure.
- Agent builders who want a single MCP server with search, scrape, and browser tools.

## Who They Miss

- Local-first users who want browser actions to happen inside their own logged-in browser profile.
- Teams that do not want a scraping/proxy vendor in the trust path for sensitive workflows.
- Users who need fine-grained human approvals, local screenshots, and deterministic evidence more than public-web reach.
