# Chrome MCP Server Marketing

## Audience

- Developers using Claude, Claude Code, Codex, Cursor, VS Code, or other MCP-capable clients.
- Users who want AI assistants to operate in the browser they already use.
- Open-source adopters who prefer local tools over hosted browser platforms.

## Positioning

- Core claim: turn Chrome into an intelligent assistant that AI can control.
- Differentiator: the product uses a Chrome extension so the agent can work in the user's real browser environment instead of launching an isolated Playwright browser.
- Breadth claim: 20+ tools, semantic search, cross-tab context, screenshots, network monitoring, bookmarks, history, and visual editing.

## Customer Model

- MIT-licensed open-source project.
- Adoption is driven by GitHub, npm/package install paths, third-party MCP registries, and developer community sharing.
- No visible commercial pricing model in the captured public surface.

## Who They Capture

- Developers who want transparent local browser control without a cloud browser bill.
- MCP users who need a broad toolset and can tolerate setup complexity.
- Bilingual/open-source communities that benefit from English and Chinese documentation.

## Who They Miss

- Enterprises that need packaged policy, audit retention, SSO administration, and support contracts.
- End users who want a polished AI browser with clear task history and recovery UX.
- Teams that need high-scale hosted browser pools, proxy/CAPTCHA handling, or managed concurrency.
