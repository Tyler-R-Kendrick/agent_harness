# Claude Computer Use Marketing

## Audience

- Developers building agents that need GUI interaction.
- Enterprises experimenting with automating legacy apps, browser portals, and back-office software.
- AI infrastructure vendors that want to wrap a strong model-owner computer-use primitive.

## Positioning

- Core frame: Claude can use computers the way people do.
- Anthropic positions the feature as a model capability with explicit safety caveats.
- The pitch is not a consumer browser; it is a foundation for agent products.

## Customer Model

- API usage and enterprise agreements.
- Integrators pay Anthropic for model calls and monetize their own product layer.
- The reference app lowers adoption friction but does not replace a production UI.

## Who They Capture

- Teams that need GUI automation beyond browsers.
- Developers who prefer model-provider primitives over full-stack agent vendors.
- Enterprises that want to own the sandbox, approvals, and audit surface.

## Who They Miss

- End users who want a ready-to-use browser assistant.
- Buyers who need deterministic browser automation without building infrastructure.
- Teams unwilling to assume computer-use security risk.
