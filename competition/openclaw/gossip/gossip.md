# OpenClaw Gossip

## Positive Signals

- The project has strong public attention because it promises a personal assistant that works through familiar messaging apps.
- Self-hosting and local-device control resonate with users who distrust fully hosted assistant platforms.
- The broad channel list makes the product feel useful beyond a single browser or desktop app.
- External guides and news coverage indicate OpenClaw has become recognizable outside a narrow GitHub audience.

## Negative Signals

- Public reporting found malicious third-party OpenClaw skills on ClawHub, including cryptocurrency-themed malware and infostealer behavior.
- Security coverage repeatedly notes that skills can involve executable local code, file access, shell commands, and network access.
- The product's former names and fast renaming history make impersonation and search confusion easier.
- A messaging-first assistant can perform consequential work without the user seeing a rich browser trace unless the product makes evidence explicit.

## Bug And UX Complaints To Track

- Skill provenance and moderation.
- Local execution permissions and sandboxing.
- User education around copy-pasted terminal commands.
- Credential and browser-password exposure from malicious skills.
- Auditable evidence for actions performed from chat commands.
