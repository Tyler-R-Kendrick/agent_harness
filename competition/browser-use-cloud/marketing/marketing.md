# Browser Use Cloud Marketing

## Audience

- Developers building AI web automation and browser-control agents.
- Startups that want managed browsers, stealth, CAPTCHA solving, residential proxies, and profiles.
- Teams that want to graduate successful agent workflows into deterministic APIs.

## Positioning

- Core claim: AI browser automation with stealth browsers, CAPTCHA solving, residential proxies, and managed infrastructure.
- Market frame: browser automation should be accessible from one SDK and scalable across agents, raw sessions, and skills.
- Product promise: use AI for quick tasks, CDP for custom automation, Skills for repeatable production endpoints.

## Customer Model

- Pay-as-you-go credits plus subscription tiers.
- Pricing includes task init, model-step cost, browser-session time, skill creation/execution, and proxy data.
- Open-source adoption feeds cloud conversion.

## Who They Capture

- Python/TypeScript developers who want a popular browser-agent library.
- Teams that prefer Skills/API endpoints over one-off agents.
- Automation builders who value managed stealth and CAPTCHA support.

## Who They Miss

- Users who need a daily browser UI rather than API automation.
- Teams with strict no-cloud browser processing policies.
- Buyers who need simple flat pricing.
