# Claude Code With Chrome Marketing

## Target Customer

- Claude Code users who want browser testing and debugging inside the same agent conversation as code changes.
- Paid Claude users on Pro, Max, Team, and Enterprise plans.
- Developers testing authenticated web apps, forms, redirects, console errors, and multi-tab research flows.

## Positioning

- Anthropic positions Chrome integration as a capability of Claude Code, not a separate browser-agent product.
- The message is practical: connect Claude to Chrome to test web apps, debug console logs, fill forms, and extract data.
- The paid-plan beta framing keeps access tied to the broader Claude account and subscription model.

## Customer Model

- Bundled into Claude paid tiers rather than sold as usage-priced browser infrastructure.
- Enterprise capture comes from Claude Code/Team/Enterprise adoption, where browser testing becomes another agent tool.
- Distribution advantage comes from being first-party and documented in the main Claude Code docs.

## Who They Capture

- Developers already standardizing on Claude Code.
- Teams that want a visible browser surface without installing community MCP bridges.
- Users who need logged-in browser context during coding and QA loops.

## Who They Do Not Capture

- Teams that need vendor-neutral browser control across multiple LLM clients.
- Buyers that require deterministic replay, formal evidence retention, or local-first data boundaries outside Anthropic's product.
- Users blocked by beta availability, extension pairing, or organization policies against browser extensions.
