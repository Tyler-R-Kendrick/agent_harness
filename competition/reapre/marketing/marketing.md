# Reapre Marketing

## Target customer

- Automation engineers and agent builders who need a unified API for browser, desktop, Android, and app execution.
- Teams that want MCP for AI-native access but still need HTTP, WebSocket, OAuth, API keys, webhooks, recordings, and usage controls.
- Operators who outgrow a single browser runner because their workflow reaches files, installed apps, mobile surfaces, or scheduled jobs.

## Positioning

- "One control plane for AI" is the core frame.
- Reapre positions browser automation as a starting layer, then expands into computer control, mobile automation, app connectors, templates, policies, and replay.
- Pricing is request-volume based rather than browser-hour based: free starts at 100 requests/month, Pro at $29/month, Enterprise at $99/month.

## Captures

- Buyers who think in governed operations, API keys, dashboards, retries, webhooks, and audit trails.
- Teams that need mixed UI/API/device automation in one interface.
- Developers who want MCP without giving up standard hosted-service integration patterns.

## Does not capture

- Users who want local-only browser control with no cloud relay.
- Buyers who only need lightweight local Chrome evidence and do not want a broad automation platform.
- Security-sensitive teams that are uncomfortable with one vendor brokering browser, desktop, mobile, and app connectors.
