# Beam

## Overview
iPad-first browser with WebKit-based rendering. Sidebar-first interface with command palette for navigation. Beam Intelligence provides agentic chat capabilities with tab context. Multi-model support (Apple Intelligence + cloud models). Niche but viable positioning in tablet market.

## Technical Approach
- **Base**: WebKit
- **AI Integration**: Beam Intelligence agentic chat with tab context awareness
- **Key Differentiator**: iPad-first design; sidebar-first UX; command palette navigation

## Key Learnings
- Tablet-optimized interface requires fundamentally different UX than desktop
- Sidebar-first paradigm is natural fit for narrower screens
- Command palette reduces menu navigation friction
- WebKit-based allows native iOS/iPadOS integration advantages

## UX Innovations
- Sidebar-first layout optimized for tablet screens
- Command palette for keyboard-driven navigation
- Beam Intelligence with tab context awareness
- Apple Intelligence + cloud model flexibility
- Touch-first interaction patterns

## Risks & Concerns
- iPad market is smaller than desktop/mobile phone markets
- WebKit limitations on third-party customization vs Chromium
- Multi-model integration adds complexity
- App Store constraints on browser functionality

## Notes
Beam's iPad-first positioning is smart niche strategy. Tablet users are underserved by mainstream browsers optimized for desktop/phone. WebKit base provides native integration but limits customization. Viable business model if tablet adoption continues growing.
