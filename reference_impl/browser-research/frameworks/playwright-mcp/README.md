# Playwright MCP

## Overview
Microsoft-backed browser automation framework using accessibility tree approach instead of DOM. Launched March 2025. Designed for AI agent interaction with semantic understanding of page structure. MCP-native integration enables broad tool ecosystem.

## Technical Approach
- **Base**: Playwright (cross-browser automation)
- **AI Integration**: Accessibility tree parsing; MCP protocol for tool integration
- **Key Differentiator**: Semantic page understanding via accessibility trees; MCP-native

## Key Learnings
- Accessibility tree approach provides semantic understanding vs raw DOM
- MCP protocol integration creates broader tool ecosystem
- Microsoft backing provides credibility and resources
- March 2025 launch signals active development in automation space

## UX Innovations
- Accessibility tree output for agent understanding
- MCP-native tool ecosystem integration
- Cross-browser support (Chromium, Firefox, WebKit)
- Semantic page structure vs DOM-level details

## Risks & Concerns
- Accessibility trees can be incomplete/inaccurate on complex sites
- Dependency on Playwright maintenance
- MCP ecosystem still emerging; adoption unclear
- Market saturation with Browser Use dominance
