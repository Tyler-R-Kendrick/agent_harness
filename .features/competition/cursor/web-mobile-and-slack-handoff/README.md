# Web Mobile And Slack Handoff

- Harness: Cursor
- Sourced: 2026-05-01

## What it is
Cursor lets users start cloud-agent work from the web, from a phone, or from Slack, then hand the same run back into the desktop editor for review, merge, or takeover.

## Evidence
- Official docs: [Web & Mobile](https://docs.cursor.com/background-agent/web-and-mobile)
- Official product page: [Cursor product overview](https://cursor.com/product)
- First-party details:
  - Cursor documents its web app as a way to kick off coding agents from any device and then continue the same work inside the desktop editor
  - the docs explicitly call out team share links, pull-request review from the web interface, and Slack notifications on completion
  - Slack is also part of the launch surface, with `@Cursor` mentions able to trigger agents directly from chat
  - the main product page keeps web, mobile, and Slack as part of the cross-surface agent pitch instead of treating them as separate tools
- Latest development checkpoint:
  - Cursor's current web/mobile docs still position cross-device handoff as a primary workflow, which suggests the ambient entry-point model is strategic and ongoing

## Product signal
Cursor is optimizing for agent continuity across intake surfaces, which is a strong reminder that the winning harness may start in chat or mobile and only finish in the IDE.
