# Long-Term Memory

- Harness: DeerFlow
- Sourced: 2026-04-26

## What it is
DeerFlow keeps persistent cross-session memory about the user profile, preferences, workflows, and accumulated knowledge.

## Evidence
- Official site: [DeerFlow](https://deerflow.tech/)
- GitHub README: [DeerFlow - 2.0](https://github.com/bytedance/deer-flow/blob/main/README.md)
- First-party details:
  - the site highlights long/short-term memory as a headline 2.0 capability
  - the README says memory is stored locally and remains under user control
  - duplicate fact entries are skipped so memory does not bloat with repeated preferences

## Product signal
This reinforces that durable local memory is now expected infrastructure for long-horizon harnesses.
