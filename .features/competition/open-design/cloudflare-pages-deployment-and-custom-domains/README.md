# Cloudflare Pages Deployment And Custom Domains

- Harness: Open Design
- Sourced: 2026-05-13

## What it is
Open Design can publish generated artifacts directly to Cloudflare Pages and attach custom domains, turning agent output into a deployable surface instead of a local preview only.

## Evidence
- Official release notes: [Open Design 0.6.0](https://github.com/nexu-io/open-design/releases)
- Official README: [Open Design](https://github.com/nexu-io/open-design)
- First-party details:
  - the `0.6.0` release highlights one-shot Cloudflare Pages deployment for generated artifacts
  - the same release adds custom-domain support on top of Pages deployment
  - the README also positions the product as deployable locally, on Vercel for the web layer, and as packaged desktop apps
- Latest development checkpoint:
  - the May 9, 2026 `0.6.0` release explicitly frames the product shift as moving from endless iteration toward shipping artifacts anywhere

## Product signal
Open Design is collapsing the gap between generation and delivery by treating deployable artifact publishing as part of the harness contract.
