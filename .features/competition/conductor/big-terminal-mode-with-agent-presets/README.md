# Big Terminal Mode With Agent Presets

- Harness: Conductor
- Sourced: 2026-05-12

## What it is
Conductor has a large terminal-first execution surface that lets users work with different coding agents from one workspace shell instead of being locked into a narrow in-chat terminal panel.

## Evidence
- Official docs: [Big Terminal Mode](https://www.conductor.build/docs/reference/big-terminal-mode)
- Official changelog: [v0.50.0](https://www.conductor.build/changelog/0.50.0)
- First-party details:
  - Big Terminal Mode expands the terminal into a primary working surface
  - Conductor documents presets for external agents inside that terminal flow
  - the `v0.50.0` changelog adds a custom auto-run command for Big Terminal Mode
- Latest development checkpoint:
  - the April 18, 2026 `v0.50.0` release shows Conductor still investing in terminal-centric workflows instead of treating the terminal as legacy escape hatch

## Product signal
This suggests operators still want high-agency terminal control even inside more visual harnesses, especially when they need to mix multiple agent runtimes and manual commands.
