# Warp Drive Workflows And Notebooks

- Harness: Warp
- Sourced: 2026-04-30

## What it is
Warp Drive turns commands, notebooks, and team knowledge into reusable assets that the agent and human can both invoke inside the terminal.

## Evidence
- Official docs: [Warp Drive Overview](https://docs.warp.dev/knowledge-and-collaboration/warp-drive/overview)
- Official docs: [Workflows](https://docs.warp.dev/knowledge-and-collaboration/warp-drive/workflows)
- Official docs: [Notebooks](https://docs.warp.dev/knowledge-and-collaboration/warp-drive/notebooks)
- First-party details:
  - Workflows package repeatable command sequences with variables and team sharing
  - Notebooks mix runnable terminal blocks with explanatory text, making them closer to operational playbooks than plain docs
  - Drive assets are searchable and can be surfaced in command workflows instead of forcing users to remember hidden scripts
  - the feature set spans local productivity and shared team enablement rather than staying personal-only
- Latest development checkpoint:
  - Warp still positions Drive as a major collaboration layer, which keeps the agent grounded in reusable operational knowledge instead of pure prompt memory

## Product signal
Warp treats reusable operational docs as executable product artifacts, which is relevant for any harness that wants team workflows to outlive individual prompt threads.
