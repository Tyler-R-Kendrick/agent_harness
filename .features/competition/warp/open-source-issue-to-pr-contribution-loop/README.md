# Open-Source Issue-To-PR Contribution Loop

- Harness: Warp
- Sourced: 2026-05-31

## What it is
Warp now runs its own product development in public with Oz agents triaging issues, planning work, implementing fixes, and opening pull requests against the open-source Warp repository.

## Evidence
- Official post: [Warp Open-Sources Its Agentic Development Environment](https://www.warp.dev/newsroom/2026/4/28/warp-open-sources-its-agentic-development-environment)
- Official post: [Open-sourcing our docs and the agents that maintain them](https://www.warp.dev/blog/open-sourcing-our-docs-and-the-agents-that-maintain-them)
- First-party details:
  - Warp says public GitHub issues are now the system of record for features, roadmap, and discussion.
  - the company describes Oz as triaging issues, asking clarifying questions, generating implementation plans, writing code, and opening pull requests in the open.
  - Warp positions session links, reviews, and progress visibility as part of the contribution model rather than internal-only tooling.
  - the same public workflow is described as reusable for other open-source projects, not just Warp's own repo.
  - Warp's docs post says documentation also feeds the context layer behind Warp Agent through RAG, tying public docs maintenance back into agent quality.
- Latest development checkpoint:
  - Across the April 28, 2026 open-source launch and the May 4, 2026 docs follow-up, Warp moved from "agent-enabled product" to "agent-operated public development loop."

## Screenshots and demos
- Official visuals in the open-source launch show the ADE and Oz-driven development framing.
- The docs-maintenance post includes images and references to the public docs stack and the agents that keep it in sync.

## Product signal
Warp is productizing not just agent execution but the public intake-to-shipping loop around it, which matters because it makes governance, review, and visibility part of the harness story rather than external process.
