# Skills Library

- Harness: Codex
- Sourced: 2026-05-16

## What it is
Codex uses installable and team-shareable skills to package instructions, resources, scripts, and workflow context that travel across the app, CLI, and IDE extension.

## Evidence
- Official product post: [Introducing the Codex app](https://openai.com/index/introducing-the-codex-app/)
- Official docs: [Codex app features](https://developers.openai.com/codex/app/features)
- First-party details:
  - skills created in the app can be reused in the CLI and IDE extension
  - repository-checked-in skills can be shared with a whole team
  - the app sidebar includes a Skills area for exploring skills across projects
  - automations can explicitly trigger skills with `$skill-name`
- Examples called out by OpenAI:
  - design implementation from Figma
  - Linear project management
  - deployments to Cloudflare, Netlify, Render, and Vercel
  - image generation and document creation
- Latest development checkpoint:
  - Codex skills are now tightly linked to app discovery and automations, not only manual invocation

## Product signal
OpenAI is treating workflow packaging as a core primitive, not an advanced customization.
