# Goal Mode Outcome And Success Criteria

- Harness: Codex
- Sourced: 2026-06-02

## What it is
Goal mode lets users define an explicit outcome and success criteria so Codex can keep longer-running work aligned to the intended result across the app, IDE extension, and CLI.

## Evidence
- Official release notes: [ChatGPT release notes](https://help.openai.com/en/articles/6825453-chatgpt-release-notes)
- Official use cases: [Codex explore](https://developers.openai.com/codex/explore)
- First-party details:
  - Goal mode was called generally available in the May 21, 2026 Codex update
  - OpenAI says it works across the Codex app, IDE extension, and CLI
  - the Codex use-case library now explicitly frames "Follow a goal" as a durable objective for long-running work
  - the feature is positioned around keeping work moving toward a stated end-state rather than reacting only to the latest prompt
- Latest development checkpoint:
  - the current product story is shifting from prompt-by-prompt steering toward declared outcomes with success conditions that survive longer execution windows
