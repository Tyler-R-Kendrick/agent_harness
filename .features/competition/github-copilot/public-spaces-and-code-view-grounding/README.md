# Public Spaces And Code View Grounding

- Harness: GitHub Copilot
- Sourced: 2026-05-05

## What it is
Copilot Spaces can now be shared publicly or individually, and files can be added to a Space directly from the GitHub code viewer.

## Evidence
- Docs: [GitHub Copilot features](https://docs.github.com/en/copilot/get-started/features)
- Changelog: [Copilot Spaces: Public spaces and code view support](https://github.blog/changelog/2025-12-01-copilot-spaces-public-spaces-and-code-view-support/)
- GitHub documents:
  - public Spaces with link-based access for open-source workflows
  - individual sharing for personal collaboration
  - code-view entry points that add files into a Space without leaving the repository browser
  - RBAC and repository permissions preserved for shared/public views

## Product signal
GitHub is treating grounded context sets as shareable artifacts, not just private prompt scaffolding.
