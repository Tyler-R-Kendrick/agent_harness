# Copilot Memory

- Harness: GitHub Copilot
- Sourced: 2026-04-24

## What it is
Copilot Memory stores useful repository details and now also captures user-level preferences that can follow a person across repositories and Copilot experiences.

## Evidence
- Docs: [GitHub Copilot features](https://docs.github.com/en/copilot/get-started/features)
- Docs: [About GitHub Copilot cloud agent](https://docs.github.com/en/copilot/concepts/agents/cloud-agent/about-cloud-agent)
- Changelog: [Copilot Memory supports user preferences for Pro, Pro+ users](https://github.blog/changelog/2026-05-15-copilot-memory-supports-user-preferences-for-pro-pro-users/)
- GitHub documents:
  - repository-level memory remains available for project-specific context
  - user-level preferences can remember preferred commit style, pull-request structure, and communication style
  - personal memory settings expose review and deletion controls

## Product signal
GitHub is moving from repo memory toward multi-scope memory that blends project context with portable personal preferences.
