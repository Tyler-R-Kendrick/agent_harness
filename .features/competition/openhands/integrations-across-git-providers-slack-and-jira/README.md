# Integrations Across Git Providers Slack And Jira

- Harness: OpenHands
- Sourced: 2026-05-02

## What it is
OpenHands exposes native integration surfaces across GitHub, GitLab, Bitbucket, Slack, and Jira-oriented workflows, with different setup paths for Cloud and self-hosted/local deployments.

## Evidence
- Official docs: [Integrations Settings](https://docs.openhands.dev/openhands/usage/settings/integrations-settings)
- Official homepage: [OpenHands](https://openhands.dev/)
- First-party details:
  - the integrations settings page documents GitHub, GitLab, Bitbucket, and Slack integrations, with Jira Cloud listed in the Cloud navigation
  - OpenHands Cloud lets users configure repository access and install the Slack app
  - the homepage says users can delegate tasks directly from GitHub, GitLab, Slack, or via the API and SDK
  - OpenHands also markets native integrations with GitHub, GitLab, CI/CD, Slack, and ticketing tools
- Latest development checkpoint:
  - the current product and docs surfaces present integrations as a core product dimension rather than optional extras

## Product signal
OpenHands is trying to meet teams where work already happens. The combination of issue trackers, chat ingress, git providers, and API/SDK surfaces makes the harness feel orchestration-ready rather than app-bound.
