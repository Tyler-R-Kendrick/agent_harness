# Open SDK And Large-Codebase Coordination

- Harness: OpenHands
- Sourced: 2026-05-02

## What it is
OpenHands exposes an SDK for building custom agents and workflows, and positions it as a reusable runtime layer rather than just an app integration surface. Its SDK messaging also calls out a Large Codebase SDK that sequences changes and allows parallel agents to work without conflicts.

## Evidence
- Official product page: [OpenHands SDK](https://openhands.dev/product/sdk)
- First-party details:
  - OpenHands describes the SDK as an open toolkit for building custom agents and workflows programmatically
  - listed capabilities include programmatic task creation, custom agent behaviors, tool and API integrations, model selection and routing, and fine-grained execution control
  - the SDK FAQ says the Large Codebase SDK maps dependencies, sequences changes correctly, and lets multiple agents work in parallel without conflicts
- Latest development checkpoint:
  - the current SDK page positions OpenHands as an open, secure, scalable agent platform and highlights the coding-agent SDK as a differentiator

## Product signal
OpenHands is not only shipping an end-user harness. It is also selling the harness primitives as reusable infrastructure for teams that want to embed or extend the runtime.
