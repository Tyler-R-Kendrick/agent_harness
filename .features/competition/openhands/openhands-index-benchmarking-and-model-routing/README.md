# OpenHands Index Benchmarking And Model Routing

- Harness: OpenHands
- Sourced: 2026-05-02

## What it is
OpenHands created a continually updated benchmark called the OpenHands Index to help teams choose models based on software-engineering task type, cost, and runtime instead of relying on a single generic leaderboard.

## Evidence
- Official blog: [Introducing the OpenHands Index](https://openhands.dev/blog/openhands-index)
- First-party details:
  - OpenHands calls it a broad-coverage, continually updated leaderboard
  - it evaluates models across issue resolution, greenfield apps, frontend development, software testing, and information gathering
  - the stated goal is to let users choose models based on their priorities and use cases, including switching models for planning, implementation, or review/testing
- Latest development checkpoint:
  - the January 29, 2026 launch post positions the Index as a response to model-agnostic harnesses needing better routing and selection signals than SWE-Bench alone

## Product signal
OpenHands is turning evaluation into a product input for model choice. That suggests future harnesses will increasingly route work by task class, cost envelope, and latency target instead of treating one model as universally best.
