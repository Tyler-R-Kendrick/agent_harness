# Agent Control Plane Budgets Policies And Audit Trails

- Harness: OpenHands
- Sourced: 2026-05-02

## What it is
OpenHands is explicitly productizing a software-agent control plane. The pitch is centralized policy, routing, secrets, budgets, auth, observability, and auditability for running many coding agents beyond a single laptop session.

## Evidence
- Official blog: [The Software Agent Control Plane](https://openhands.dev/blog/agent-control-plane)
- First-party details:
  - OpenHands describes three pillars for software agents: harness, orchestrator, and control plane
  - it says the control plane centralizes LLM routing policies, MCP access, secrets access, budgets, and user management
  - the same post emphasizes centralized audit trails and policy hooks that can halt risky actions for human review
  - OpenHands also frames cloud agents as easy to trigger from schedules or events in Slack, GitHub, and Jira once they run in managed infrastructure
- Latest development checkpoint:
  - the April 3, 2026 post positions OpenHands Enterprise as a unified control-plane offering for software agents at scale and includes architecture diagrams on the source page

## Product signal
This is a direct move beyond single-user coding copilots. OpenHands is telling buyers to think in terms of fleet management, governance, and organization-wide ROI instead of isolated chat sessions.
