# Document To Skills Reusable Workflow Capture

- Harness: Kimi AI
- Sourced: 2026-05-27

## What it is
Document to skills turns high-quality documents into reusable agent skills so prior work can become an executable workflow asset instead of a dead artifact.

## Evidence
- Official model page: [Kimi K2.6](https://www.kimi.com/ai-models/kimi-k2-6)
- Official help: [What are Skills?](https://www.kimi.com/help/agent/what-are-skills)
- Official help: [Creating skills FAQ](https://www.kimi.com/help/agent/create-custom-skills-faq)
- First-party details:
  - Kimi says users can click `+` then `Skills -> Document to skills`, upload a document, and ask Kimi to extract, replicate, or turn it into a reusable skill
  - the K2.6 page says these generated skills can be applied to future tasks and combined with Agent Swarm for more consistent complex deliverables
  - the Skills help page says skills bundle working approaches, standards, scripts, tools, and references, rather than acting as a prompt snippet only
  - the same help page explicitly distinguishes direct prompting from reusable skills that auto-apply preset structure and tone
  - Kimi also advertises open-source skill hubs and says the same skill model can span Agent, Kimi Code, and Kimi Claw contexts
- Latest development checkpoint:
  - K2.6 elevates "office document to skill" from a niche creator flow into a flagship product story, making artifact-derived skills part of the main agent value proposition

## Product signal
Kimi is turning finished documents into reusable agent behavior, which closes the loop between outputs, memory, and workflow packaging.
