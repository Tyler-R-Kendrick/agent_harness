# Agent Template

Use this structure unless the user asks for something more specific.

```md
# <Agent Name>

## Purpose
- Describe what this agent is responsible for.

## Goals
- List the outcomes the agent should optimize for.

## Constraints
- List the rules, safety limits, and quality bars.

## Workflow
1. Explain how the agent should begin.
2. Explain how the agent should validate its work.

## Deliverables
- Describe the expected outputs or file changes.
```

Keep the generated file narrow. A focused agent is easier to invoke and easier to evaluate than a general-purpose persona.
